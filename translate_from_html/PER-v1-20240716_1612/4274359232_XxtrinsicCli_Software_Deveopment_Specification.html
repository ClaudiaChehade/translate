XxtrinsicCli Software Deveopment Specification

Requirements

[CNWVIII -95270] [OTA3] _ [OnlineCaib] Offboard parameter processing tool -tracker08 (bosch.com)

A tool XXTRINSICCLI, a tool that runs in the QNX command and is read and write and updated for the Sensorxxtrinsic structure in Persistency.Need support:

  • As a test car, the commercial vehicle replace the domain control (D -shaped C -like C), and the tool requires that the tool can support the reading of SensorxXtrinsic from D -like to XXTRINSIC.BIN, and write XXTRINSIC.BIN into C -like

  • In the After -Sales Changes, if there is cloud data, you need tools to support the calibration data (Rosbag) from the cloud, write it into the new domain control

  • In the After -Sales Changes, if there is no cloud data, the tool supports the data (Delta value) from the factory calibration of the upper machine, updated it into the new domain control

  • Check the intrinsic stored in Persistency and read from Camera_Provider. If the Camera_Provider fills random value and writes it to the Persistency, it will report to the fault_calib_intrinsic_inValid. Intrinsic in STRINSIC in Stence

  • In test scenarios such as HIL/HOL, you need tools to support tool support to write YAML files into the test equipment

Suppose:

  • In the after -sales changes, if there is cloud data

    • Existence sub -tool 1, convert the calibration data extracted from the cloud BAG into xxtrinsic.bin

  • In the after -sales changes, if there is no cloud data

    • The new domain control will be provided after the new domain control first, so that the internal reference and SN number are first deposited into the new domain control first

    • Exist the sub -tool 2, turn the upper machine Delta value to the Yaml file

Software Design

Development tool XXTRINSICLI.Support the following modes:

  1. MODE_READ: Read the Sensorxxtrinsic structure from Persistency, and store it as xxtrinsic.bin according to binary storage.

  2. MODE_WRITE: Write XXTRINSIC.BIN in the domain control to persistency

  3. MODE_UPDATE: Read the Sensorxxtrinsic structure from Persistency, analyze YAML, extract external information from YAML, update the Sensorxxtrinsic structure ( Only update the external part ), And write with persistency

  4. MODE_CLEAR_RANDOM: Read the Sensorxxtrinsic Structure from Persistency, and zero the non -effective bit of Sensor_intrinsics from the random value zero

  5. MODE_WRITE_YAML: Analyze the yaml file, convert it to the SensorXXTRINSIC structure, and store it to Persistency
