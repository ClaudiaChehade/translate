4. Model Design && DAG

  • Tracking currently has no mirror image, so read C ++ code to understand the details.The overall module is shown in the figure below.Two functions are Obstacle Tracking, and the other is FreeSpace Clusterizer.

  • FreeSpace Clusterizer: It is mainly a post -processing, adding a convex bag processing, and becoming an Area.

  • Obstal Tracking: The overall idea is Kalman+Hungarianalgorithm, And there is no CASE branch.

    • CostMatrix: Main Location+Shape Similalic, Location is divided into distribution of X and Y directions and iOU. Shape is mainly width and length. Do not consider Height. Take the maximum value of the two.The maximum value of the threshold.

    • Kalman: The main motion model is also a conventional acceleration model x1 = x0+vt+1/2at*t, there is in the observation amount VX and VY (Output of upstream models), and the estimated value does not have any CASE branch.

    • HUNGARIANALGORITHM: Matching a two -point diagram, a classic algorithm.

  • Overall experience: PIPELINE is relatively simple, without any case, " Clean and tidy ”。
