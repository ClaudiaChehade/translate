HVM Detailed Design

HVM is a platform code for integrated W1 Driving.The detailed design of the HVM below mainly introduces the detailed code design of the integrated part.


one.introduce

HVM: Host Vehicle Movement.This module provides more accurate vehicle dynamic results in the driving scene.

The input of this module is the vehicle CAN signal + vene Parameters + Vehicle Specific Parameters:

  • Wheel speed.

  • Sensor yaw rate.

  • Longitudinal acceleration sensor.

  • Lateral acceleration sensor.

  • Steering wheel angle.

  • Vehicle specific parameters.

  • ESC state.

  • …

The output of the module is:

  • Current trajectory.

  • Wheel angle

  • Longitudinal acceleration sensor / road slope estimation

  • Lateral acceleration.

  • Yaw rate.

  • Oval course detection.

  • Wheels standstill.

  • Vehicle Parameter (e.g. wheel base, wheel track).

The functional security level of this module is ASILB.

Figure 1 HVM software architecture

For example, the software architecture map: PME module obtains vehicle CAN signal and other vehicle parameters from the outside, as well as Persistent Data (the calibration results during the driving process of the vehicle, such as steering wheel corner+front wheel corner, etc.)Calculate the results of the learning, and then output the dynamic results of the vehicle and continue to optimize and store the Persistent Data.

Among them, the HVM part is the adaptation layer code. The function of this layer code has the following points:

1. Convert the CAN signal of different models to the universal signal parameters required for the PME layer

2. Monitor the signal layer of the CAN signal layer and identify bad operating conditions in advance

HVM_PME Static Design Module is as follows of red frames:

2. The left end of thepME is the SPPHVM layer adaptation code.

The main CPP files are SPPHVMRunnable.This layer adaptation code is mainly responsible for converting vehicle parameters, signals, and Persistent Data into universal functions of the platform core relationship layer.

5 code reuseability

PME part is reused from the Wave1 platform code.The PME platform is a platform for the use of Bosch Driving function to provide self -car positioning and vehicle sports. Theoretically, it can be directly used in the W3 Driving function, so in the W3 project, the PME platform is reused.

SPPHVM is a glue layer code. Data conversion and mapping work for different projects and models of different items and different projects cannot be reused.
