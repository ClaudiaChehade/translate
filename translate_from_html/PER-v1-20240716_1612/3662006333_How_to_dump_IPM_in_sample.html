background

To quickly test the stitching diagram, verify the calibration of the fish eye.


step

  1. Normal script startup software (the software must be with _debug_ words, indicating that IPM is activated output)

  2. You need to open another terminal that is still the CD to the OUTPUT folder. First of all, the environment variables are activated. Remember that there is a "." Point and space in front

. ./setup_runtime_env.sh

3. Activate the terminal of the environment variables above, enter the/output/bin folder, record Bytesoup by MTA RAW, you can use the following naming IPM (generally you can record the function output of the parking space detection and verification)

./mta_raw_output_gateway -e -w MTA.* \
.*viper_parking__debug_stitched_img.* \
.*viper_parking__xper_viper_parking_slot_senderport.* \
-r 2048


4. After running successfully, you will get stuck in an waiting interface. You can cancel CTRL+C at a time of approximate time. Save a Bytesoup file with time under/output/bin, copy it to the upper machine (Ubuntu system) (Ubuntu system)Come up.

5. A small tool provided by MW colleagues can be transformed into Rosbag directly in the Ubuntu system.The tool link I put in \\ bosch.com\dfsrb\DfsCN\DIV\XC\Engineering\domain\WAVE3\00_DataExchange\donny\mta_to_ros\mta_20230920.zip

Copy it to the Ubuntu upper machine and decompress it.

6. Put the bytesoup in Step 4 to the MTA_20230920 folder and run the following commands. For example, this bytesoup package is called IPM1.BYTESOUP. You are ready to save a ROSBAG with the same name.

./toros.sh ipm1.bytesoup ipm1.bag

A IPM1.bag file will be turned out of the out folder.You can use Foxglove to open the IPM diagram directly.
