Real car debugging process after PAD access

Important information is shown bold in red words, please be sure to pay attention

  1. Essential hardware equipment

    • Android PAD or Android phone (referred to as PAD)

    • One adapter at Typec to the network port

    • One router (installed on the car)

  2. Environmental requirements

    1. Full PAD in advance before getting on the car, and some APK installed in the car consumes very powerful power.

    2. Pad connects the adapter of the Typec to the network port, and then the network cable is connected to the car router on the car

    3. The IP address assigned by the router DHCP is fixed at 10.43.0.19, Make sure that the IP address is fixed to 10.43.0.19 after PAD access (if you do n’t know how to operate at this step, please find a colleague of PMT YANG Loko (XC-AS/PJ-WIN-PMT1) assist)

      1. For specific operation steps, please refer to the document: 04_How to connect Pad with ECU

  3. The operation of the Internet Unicom on the real car ADCC:

    1. ssh root@10.43.0.28 (log in Master SOC)

    2. Start the algorithm (EG. SH Start_Master_driving) according to the normal process, wait for the algorithm to start normally

    3. Turn off the firewall, instruction: PFCTL -P/SOCK4/DEV/PF -D (must be executed to ensure that the command is not wrong)

    4. Switch SOCK, instruction: Export Sock =/SOCK4

    5. Confirm the pad, instruction: ping -n 10.43.0.19 (optional, just to confirm whether you can ping through)

    6. Close the wifi of PAD , Start the AOM APK on the pad, and see the blue sky and white clouds, it means that the startup is successful

    7. Enter the command on the master SoC: NetStat -An | GREP 51011, ensure that the return status value is ESTABLISHED (ESTABLISHED represents PAD and SOC network connecting.The step is not successful)

  4. PAD sending route -related operations on the real car

    1. Open the phone hotspot, turn on the pad wifi, and let the PAD connect to the hotspot of the mobile phone , Then open the Gaode Map APK

    2. Gaode Map APK has no positioning function, If the starting point is inconsistent with the actual situation, you need to use Autohelp APK to modify the actual starting point position ：

      1. Open the Autohelp map interface, move to the location of the self, click on the target location, the red mark appears, and click "Position here" in the lower right corner;

      2. Back to the map of Gaode, if the location of the car does not return to the position set in the previous step, restart the map of Gaode;

    3. Establish a destination on the Gaode Map APK (just like using Gaode Navigation/Baidu Navigation on a mobile phone or on the car). Normally, after clicking the "Starting Navigation" button, the ADCC can receive the route and display the map and planTrack, If you can't see it, drive the car along the plan for a while


Precautions:

  • Make sure that the status value of Established can be seen normally. If you can't see it, please confirm which step in front is not completed.

  • The starting point position is not strictly consistent with the actual location of the self -car, but to ensure that the location of the self -car driving is on the planned trajectory, otherwise 4.c may not see the map and planning trajectory

  • If you click "Starting Navigation" for a while, if you want to navigate ADCC again, but you can't see the new map, restart the Gaode Map APK

  • If you restart the SOC, all steps in 3 and 4 need to be re -operated

  • If you encounter that step, you can't reach it, try to restart all the equipment once again

  • AOM APK will upgrade with the domain control software version. When you need to upgrade, the map team will uniformly notify the upgrade. Reference to the available APK download path required for upgrading: HDMAP overall link joint environment dependencies
