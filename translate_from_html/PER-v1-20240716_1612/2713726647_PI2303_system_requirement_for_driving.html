SysRS_HighwayE2E_180

If ego is approaching a construction zone with current lane which will be blocked and it is detected to be away less than #P_HW_StaticObject_TorDist meter, system shall request driver to takeover the vehicle and in the mean time, keep in lane driving and slow down the vehicle with a minimum absolute value of deceleration rate of #P_HW_StaticObject_Comftax .
Note1: the p_Highway_MinDist_StaticObj is defined based on the system reaction time + distance run by ego in average lane change time with comfort deceleration. If ego is not able to detect construction zone before that distance which means there is no suffisant safe distance for lane change to avoid the construction zone, a takeover request must be sent out immediately.
Assumptions for calculation of min. distance :
- System reaction time : which means the time that system needs to 'perceive' and 'recognize' the situation + sent out the deceleration request = 0.5s
- Brake pressure built up time = 0.5s
- Average lane change procedure time : 6s
- Constant deceleration during the lane change procedure = 2.5m/s2
Note2: For the case when the construction zone is detected before reaching that min. distance, please find correponding system behavior in next chapter.
