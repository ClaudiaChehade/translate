Sorted

For all sensor data, first summarize as a file, and then sort according to TimeStamp according to each line in the summary file.

  • Type: sensor type

  • TimesStamp: The system at time stamp of the system, the system timestamp of the sensor message receiving, if there is no UTC_TIMSTAMP

  • UTC_TIMSTAMP: UTC timestamp, the timestamp after synchronization, if not, use Timestamp

  • ID: sensor ID, used to distinguish multiple sensors in the same type

  • SUB_ID: Different types of data that belongs to a certain sensor, such as images, images = 0, semantic = 1, object = 2
