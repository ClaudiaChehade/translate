How to change stitch parameters for IPM

Modify Stitching Parameters, you need to modify the software parameters in the board.but Because the/asw/folder is only readable, the stitching parameters cannot be modified directly under the/asw folder.Need us First of all, copy the/asw/folder to the readable and readable directory/cache/update/directory+adjustment of the configuration script of the eSME （ Look at this Guideline Step1 How to replace test binaries in board ), And then in The following modifications are performed in the configuration parameter file in/cache/update/directory.


First Confirmation model Is it E03 or E0Y? It is better to judge if the real car is better. E03 is a car and E0Y is an SUV.If HOL, you need to see which car you are released, and then change the corresponding parameter value. Restart and take effect


The specific parameter path is/cache/update/asw/yaaac_parameter_instances

Splicing

The file to be modified is parking_per_algo_conf_parameter_instance , This file control algorithm -related overall way.

e0y_dw_section_type = 3 # si-unit [n.a], range [0..3], 0/1/2/3 for different section types
e03_dw_section_type = 2 # si-unit [n.a], range [0..3], 0/1/2/3 for different section types

The above two values ​​control the Splinting of the two models of E0Y and E03, respectively. The value is 2 in the value of 0 °, and the value of 3 is barbell stitching.

Now the default E0Y is a barbell type, E03 is 0 ° stitching.

The other values ​​are explained as follows, 0 is 45 ° oblique angle, and 1 is 90 ° after 0 °.

Stitching parameters

The file to be modified is parking_per_veh_conf_parameter_instance， This file controls the specific parameters of the stitching, and is divided into two sets according to the model.

e03_car_box_margin_front = 0.035 # si-unit [n.a], range [-1.0..1.0], car body margin calibrate configs: front
e03_car_box_margin_right = -0.05 # si-unit [n.a], range [-1.0..1.0], car body margin calibrate configs: right
e03_car_box_margin_rear = 0.02 # si-unit [n.a], range [-1.0..1.0], car body margin calibrate configs: rear
e03_car_box_margin_left = -0.05 # si-unit [n.a], range [-1.0..1.0], car body margin calibrate configs: left
e03_mm_per_pixel = 23.4375 # si-unit [mm], range [0.0..100.0], how many mm for each pixel
e03_front_cam_extend = 0.5 # si-unit [n.a], range [-1.0..1.0], E03 front camera extend ratio
e03_rear_cam_extend = 0.3 # si-unit [n.a], range [-1.0..1.0], E03 rear camera extend ratio
e0y_front_cam_extend = 0.08 # si-unit [n.a], range [-1.0..1.0], E0Y front camera extend ratio
e0y_rear_cam_extend = 0.2 # si-unit [n.a], range [-1.0..1.0], E0Y rear camera extend ratio
e0y_car_box_margin_front = 0.03 # si-unit [n.a], range [-1.0..1.0], car body margin calibrate configs: front
e0y_car_box_margin_right = 0.05 # si-unit [n.a], range [-1.0..1.0], car body margin calibrate configs: right
e0y_car_box_margin_rear = 0.0 # si-unit [n.a], range [-1.0..1.0], car body margin calibrate configs: rear
e0y_car_box_margin_left = 0.05 # si-unit [n.a], range [-1.0..1.0], car body margin calibrate configs: left
e0y_mm_per_pixel = 23.4375 # si-unit [mm], range [0.0..100.0], how many mm for each pixel


The specific parameter explanation is as follows

E0X_CAR_BOX_MARGIN_XXXXX is a regional design that is hidden (black) hidden in the intermediate body part. For example, the front of the E03 car is 5 meters, E03_CAR_BOX_MARGIN_FRONT = 0.035 is the black border to cover the body 5*0.035 = 0.175 meters.analogy.

e0x_mm_per_pixel is the actual ground length represented by each pixel

E0X_XXX_CAM_EXTEND is the unique parameter of "barbell -type", which expands the proportion of the front or rear camera parts
