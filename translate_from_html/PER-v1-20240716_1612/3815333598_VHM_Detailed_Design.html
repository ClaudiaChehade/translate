VHM Detailed Design

Because VHM is a platform code for integrated W1 Parking.The detailed design of VHM below mainly introduces the detailed design of the integrated part.

1. Main functions

VHM, the full name is Vehicle Model.This module provides more accurate mileage meters and vehicle dynamic results in the parking scenario.

VHM input is the information of other ECUs inside the vehicle. It does not need to rely on the front -loading to locate the sensor, which can obtain more accurate positioning and dynamic results.

This module is a functional security module with a security level as ASILB.

Figure 1 VHM mainly input, output, and internal main algorithm flowchart

The main algorithm flowchart of VHM is shown in Figure 1.

Enter the matching parameters and algorithm parameters required for VHM on the left.Including the characteristic steering curve (at different angles, the curve from three polynomial fits steering wheel to the rotary angle.), The effective wheelbase, front wheels and rear wheels, the length of the wheel pulse, and the drive wheel.

The upper input is the CAN signal required by VHM, see the interface below.

The number of inputs in the lower side includes the number of wheels rotating a circle, RA S (rear axle steering) Find Table (if it is a vehicle that can be turned around the front and rear wheels).

The output on the right includes the driving distance of each cycle, the position of the position of each cycle, and the angle changes of each cycle.

The main modules include:

1. Driving distance calculation module.The amount of changes in each cycle of wheel pulse per cycle multiplied by the length of each wheel pulse to obtain the driving distance of each cycle vehicle.

2. Calculate the direction of the vehicle.The main idea is to vote for the driving direction of the 4 wheels, and the four wheels vote as the final driving direction that is determined.

3. YAW angle calculation.Use the monorail model to calculate the YAW angle when a larger steering, use the dual -track model to calculate the YAW angle when smaller steering, and use the mixed monorail and dual -track calculation YAW angle when the middle steering is used.

4. Self -calibration module.Because VHM requires high accuracy, in different scenarios of vehicles, VHM will collect related data to calibrate the proportional coefficient of Yawrate's Office, the turn of the steering wheel angle, and the four wheels.

5. Signal pre -processing.When the frame loss and the system are unstable, you can use the wheel pulse signal of the loss of the interpolation of the speed and the speed signal to overcome the bad conditions.

3. VHM's dynamic design

3.1. Status chart

Figure 3 VHMPROVIDER status chart

VHMPROVIDER's status machine has two states: Vhmhandler_oninit and VHMHANDLER_ONUPDATE.When the system is turned on, first enter the VHMHANDLER_ONINIT state. If the initialization is successful, enter the VHMHANDLER_ONUPDATE state.After that, VHMPROVIDER receives relevant data for processing every 10ms and sends the calculation results.

The sequential charts in the following are different from the two different states of VHMHANDLER_ONINIT and VHMHANDLER_ONUPDATE, respectively.

AutoCalib: The full name is AutoCalibration, which is a self -calibration module in the RBPVHM.It is used for offsetting angle, tire ratio coefficient, yawrate migrants in the steering wheel rotation angle in RBPVHM.Because the class needs to store the calibration results from time to time in the EEPROM in the system, it is necessary to define the storage method and storage path in the AOS layer.

When the RBPVHM and Valin are not initialized, the configuration word (E03/E0Y) sent by the system to the model is required, and the VHMPROVIDER combined with the model configuration to read the system parameters, initialized Valin and the RBPVHM.After the type Valin and the RBPVHM have been initialized, this step is no longer performed.

After the type Valin and the class RBPVHM are initialized, the following steps are performed:

  1. Process the data of ACU2.When the input message queue exists in ACU2 data, process the received data received and put it in the Valin queue.Call interface: Static Void Handleacu2sig (Const Canrouterchery :: Acu2 &).

  2. Process the data of ACU3.When the input message queue exists in ACU3 data, process the received data received and put it in the Valin queue.Call interface: Static Void Handleacu3Sig (Const Canrouterchery :: Acu3 &).

  3. Process the data of onebox1.When the input message queue exists in onebox1 data, process the data received by the one, and put it in the Valin queue.Call interface: Static Void HandleoneBox1Sig (Const Canrouterchery :: Onebox1 &).

  4. Process the data of onebox4.When the input message queue exists in onebox4 data, process the data received by the OneBox4 and put it in the Valin queue.Among them, the timestamp of each frame will be checked in this function. If there is a problem of retreating when there is a timestamp, it will be discovered in time and reports the problem in Send Degrade Data.Call interface: Static Void HandleoneBox4Sig (Const Canrouterchery :: Onebox4 &).

  5. Process data of SAS1.When the input message queue exists in SAS1 data, process the data received by the SAS1 and put it in the Valin queue.Call interface: Static Void HandlesaS1Sig (Const Canrouterchery :: SAS1 &).

  6. Process data of VCUCOM10.When there is VCUCOM10 data in the input message queue, process the data received to VCUCOM10 and put it in the Valin queue.Call interface: Static Void HandlevCucom10Sig (Const Canrouterchery :: vcucom10 &).

  7. Process data of PMACTLMNGR.When PMACTLMNGR data exists in the input message queue, the status machine data of the PMACTLMNGR is processed.Determine whether the parking status is converted from Searching to Guidance.If so, the RBPVHM module needs to perform corresponding coordinate system jump and conversion processing. Calling interface: Static Void Handlepmactlmngr (Const FCT_PMACTLMRGR_OUT_IF :: cpmactlmRGRGRGRATPUT &).

  8. Update Valin data.Receive the CAN data put on the Valin queue for a single scheduling for the corresponding processing.Including unit conversion, CAN signal packaging to the corresponding structure.Call the interface: void updateValinAll_data()。

  9. Valin data is converted into VSM.The data structure stored in Valin is encapsulated into an input interface of RBPVHM, which is VSM.Call interface ：void rbp_vsm_valinAllConversion_vd()。

  10. Class RBPVHM algorithm execution.The main logic of the RBPVHM algorithm.Enter the Valin data to perform corresponding processing, and calculate the POSE and vehicle sports results of each cycle.For details, see 1. The introduction part.Call Port: void RBP_VHM_COMMON_RUN10MS_VD ().

  11. Send VHM data.Output the results of each cycle calculation. Call the interface: Void Prepareoutput (XLOC :: CVHMOUTPUTINTERFACE &, UINT32_T). The interface function will call: void RBP_VHM_GetMonitordata_VD (algvhmoutput.rbp_type_vhm_monitordata_st &) and void RBP_VHM_COMMON_GETVHMOUTInterfaceData_b (ALGVVV Hmoutput.rbp_type_vhm_common_outputdata_st &).Get data from RBPVHM.

  12. Send downgrade data.Determine whether the RBPVHM has a downgrade.And send your own state to the state machine.The current supported FID includes:: VHMWicFailure、VHMOutputCheckFailed、VHMCRCCheckFailed、VHMTimestampCheckFailed。 Call the interface: Void PrepareoutputDegrade (degrade :: xlocfaultEventList &). Among them, the function will be performed:

    1. Get VHMOUTPUTCKECKFAILILD and VHMWICFAILURURIRIRIDE from the VHM algorithm warehouse. If there is a problem, a downgrade signal will be sent.

    2. This function will get the result of checking the timestamp in the Handle Onebox4 Data. If there is a problem, a downgrade signal will be sent.

    3. This function checks the results of the CRC verification when VHM Autocalib storage data. If there is a problem, a downgrade signal will be sent.
