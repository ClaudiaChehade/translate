Parking - How to integrate parking requirements into Viper and Fusion?

Requirement from parking


In short, additionally requirements from parking are

  • Specific objects need to be detected:

    • pillar, lock, stopper, charger, speed bump, road blocker, fence

  • Static objects need Bounding Box 3D + Polygon outline

Solution options

Viper

Viper can be shared by driving and parking:

Besides object detection on normal views, one top view perception pipeline shall be created if some parking features requires, e.g. parking slot detection

Fusion

There are two options:

Option 1:

Integrate central fusion of driving.

Dynamic world can be shared. However, dynamic world is not fused with USS as what parking is doing now.

Another parking world can be created to provide object contours for parking usage.

Option 2:

Independent fusion for parking

OPL: what middleware will parking use? Is it possible to interact with AOS?


Conclusion

SW architect has decided to take option2. WU Ye (XC-AS/PJ-WIN-PER2) HAI Aicheng (XC-AS/EPW2-CN) , please update your SW architecture accordingly.
