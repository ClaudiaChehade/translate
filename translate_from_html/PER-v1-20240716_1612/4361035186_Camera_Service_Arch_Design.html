Camera Service Arch Design

Architecture

Function Architecture

For J6E -specific platforms, Camera Service should have the following functions, and the internal division of the sub -module is as follows. Camera Service design is temporarily considering the universality of other platforms.

Module name

Function

Vio Driver

1. Initialize VIO configuration (vio.json/cim.json)
2. Initialize each road camera (according to Camera_x.json)
3. Support hardware failure diagnosis

4. Support hardware status query

5. Support internal reference, SN read
6. Packaging VIO -related interface

Image Encoder

1. Support to encode the multi -way input image (H264/JPEG)
2. Support the internal number of bosch mini/code flow upload.

Image Decoder

1. Support the multi -way back irrigation image (H264/JPEG)

Subscriber

Subscribe to external data stream

Diagnostic

1. Provide Error Handler framework for each submissive
2. Support fault information collection, debonce, report to report

ImageQueueManager

1. Support multi -way camera timestamp alignment
2. Support numbers to use special time stamps to logic logically logic requirements
3 Responsible for storing image pointers, cache size, image life cycle management

Iceoryx Publisher

1. Support publishing image messages (HBMEM +IceoryX)

Iceoryx Subscriber

1. Support accepting image messages (HBMEM+IceoryX)

Camera Service Manager

1. Support the entire Camera Service Function, responsible for calling the interface of each module
2. Responsible for the creation of various threads, as well as the life cycle
