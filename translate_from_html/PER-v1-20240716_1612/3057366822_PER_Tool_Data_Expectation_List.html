HOL

  1. The automated testing script has not been completed yet, so it cannot be used remotely via remote desktop for now. For example, manual reset is required for A-sample power recovery operation.

  2. The current process involves first performing data replay, and then launching start_run.sh for testing. Therefore, the data before start_run.sh starts running is not simulated and tested. For example, for a 10-minute data, only the last 9 minutes may undergo simulation testing, while the first 1 minute of data is not simulated and tested.

  3. Every time a simulation test is completed for one segment of data, it requires the complete operation process including of resetting the power of A-sample, which is terrible.

  1. It can perform automated testing with just one click, saving a lot of repetitive command-line outputs and operations. It supports the use of remote desktop to complete all steps, including A-sample power recovery operation.

  2. The order of each operation step can be changed. Even if the sequence of operations such as camera streaming, data replay, and executing start_run.sh can be changed, which will not cause the simulation test to fail.

  3. After each simulation test is completed, the repeated testing can be performed without the need to reset the power and go through the entire operation process every time.
