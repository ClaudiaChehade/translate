Introduction & tag log

Use (used side) as a conan package

  1. self.requires("w3_trans_helper/0.0.2@pjw3/dev")

  2. Including header files #include "transform_interface.hpp" in your own code

  3. Other configurations of the same conan package

Source code Compile and update (maintenance side)

  1. Set environment variables, set up setting according to your actual Conan environment installation address
    export CONAN_USER_HOME="/home/inz2szh/zhangyichi/conan_install"

  2. Compile
    cd build
    cmake ..
    make -j4

  3. Modify the TAG number in the conanfile.py

  4. Update the conan package
    ./export_upload_conan.sh

  5. It is best to update the source code corresponding to the same TAG number

How to use the library

  1. Source code location: Browse PJW3_ALG / xcalib-eng - Source Code 01 (bosch.com)

  2. docupedia Location:

  3. Configure the storage position of the calibration file (in order to prevent errors from configuration to an absolute path)
    xcalib::CTransformInterface::setCalibrationDataPath("../config/");

  • Standard file finding path: \ bosch.com \ dfsrb \ dFSCN \ DIV \ XC \ ENGINEERING \ Wave3 \_Per \ 05_maploc \ 00_Format \ dongshahuhu

  • The calibration file only needs to be related to yourself, not necessarily complete. For example

  • BEV is special. If the required change involves BEV, the following files are necessary:
    （1）nrcs_front.yaml
    （2）nrcs_right.yaml
    （3）nrcs_rear.yaml
    （4）nrcs_left.yaml
    （5）body_extrinsic.yaml
    （6）config.txt
    Config.txt format has been provided in test_calibration_lib/config. The Fisheye_size (Fish Eye Original Picture Size), Atlas_SIZE_2D (BEV Pictures), MM_PER_PIXEL (BEV represents the actual millimeter).

  • If you want to cast the ground point on the image to the BODY system, you do not provide a direct transformation at present, please follow the order of Pixel-> Bev first, and then bev-> body.

  1. Specific function calls in the way to imitate the method in test_transform_lib.cc, that is, xcalib :: ctransformInterface :: Getinstance (). Your_function ()

  2. For some basic needs, and call methods can be referred to: w3_trans_helper API - wave 3 development - Docupedia (bosch.com)

update record

  • tag: 0.0.3
    Update date: 2023.02.03
    New interface 1:
    bool getTransform (EReferenceFrame f_src_frame, EReferenceFrame f_dst_frame, const CEULER_ORDER& f_order, Eigen::Vector3d &f_euler, Eigen::Vector3d &f_translation);
    effect:
    When returning to the coordinate system, the posture is provided in the form of Euler Corner. The order of f_euler is the same as F_ORDER, that is, if f_order = zxy, f_euler = [rot_z, rot_x, rot_y]
    If it is returned to FALSE, the parameters are wrong, or the relevant parameter file is not read
    New interface 2:
    EINTRINSIC_TYPE getCameraIntrinsic (EReferenceFrame f_frame , Vector &f_intrinsic)
    effect:
    Get the internal reference of the camera and return it in f_intrinsic fixed to 30. F_Frame is all Camera_* or Camera_pixel_*, please analyze f_intrinsic according to Eintrinsic_type. The agreement is as follows.
    (1) UNKNOWN: Obtain failure
    （2）PINHOLE：k1 k2 k3 p1 p2 fx fy cx cy
    （3）FISHEYE：k1 k2 k3 k4 fx fy cx cy
    （4）OCAM：c d e cx cy poly[5] inv_poly[20]
    （5）MEI：xi k1 k2 p1 p2 fx fy cx cy
    New interface 3:
    bool undistortPixel (EReferenceFrame f_frame, const Eigen::Vector2d& f_dist_point, const double& f_undist_fx, const double& f_undist_fy, const double& f_undist_cx, const double& f_undist_cy, Eigen::Vector2d& f_undist_point);
    effect:
    Specify the internal reference and real images of the virtual camera and the pixel coordinates on real images, and remove the pixels
    New interface 4:
    bool distortPixel (EReferenceFrame f_frame, const Eigen::Vector2d& f_dist_point, const double& f_undist_fx, const double& f_undist_fy, const double& f_undist_cx, const double& f_undist_cy, Eigen::Vector2d& f_undist_point);
    effect:
    Specify the internal reference and virtual images of the virtual camera and the pixel coordinates on virtual images, and distract pixels to real pictures
