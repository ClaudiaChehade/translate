DEMO generation reference path

Method 1

The work controller on the car is running directly.

1. Copy all the files in this directory to the industrial control machine on the car, \\ bosch.com \ DFSRB \ DFSCN \ DIV \ XC \ Engineering \ Domain \ Wave3 \ 00_DataExchange \ To_ver (excluding the folder of TO_VER, only copy the file inside)

Suppose it is copied to/mnt/8t_2/demo2/exe directory.

2. CD to copy directory.

cd /mnt/8T_2/demo2/exe

3. If this Terminal has not executed the following commands, it needs to be executed first. If it has been executed, it will not be executed.

export LD_LIBRARY_PATH=$LD_LIBRARY_PATH:$PWD

source /opt/ros/noetic/setup.bash

Give ./generatepath and ./convert_path weighted (777)

sudo chmod 777 ./generatePath

sudo chmod 777 ./convert_path


The following two steps are required for each generation path.

4. Generate path.

./generatepath/mNT/8t_2/Demo/demo_03_2024-04-23-10-52-04.bag (this BAG path is replaced by the BAG path recorded on the car)

After the execution of the above command, an orp.txt file will be generated in the current directory, and this file will be given to Planning. They will generate a global_path_smoothed.txt file (if they are not called this name, change it to this name)

Copy global_path_smoothed.txt to the current directory.


5. Conversion path.

./convert_path

HPA_MAP folders will be created in the current directory, and then there will be a reference_path.yaml file.
